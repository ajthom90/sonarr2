# Milestone 11 — Sonarr v3 API Compatibility

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task.

**Goal:** Implement the Sonarr v3 REST API surface so existing clients (Overseerr, Jellyseerr, Prowlarr, LunaSea, nzb360, Notifiarr, Home Assistant) can talk to sonarr2 with zero changes. After M11, pointing an external tool at `http://sonarr2:8989/api/v3/` with the API key "just works."

**Architecture:** `internal/api/v3/` holds one handler file per resource. Each handler translates between our domain types and Sonarr's exact JSON shapes (field names, casing, nested objects). Pagination uses Sonarr's page/pageSize/sortKey/sortDirection convention. Auth via `X-Api-Key` header or `?apikey=` query param.

**Reference source:** Golden response fixtures captured from the user's live Sonarr instance at `https://sonarr.home.njathome.net` (credentials in `.env.development`). CI includes a schema-diff step against these fixtures.

## Task 2 — API key auth middleware

Add authentication middleware that checks the `X-Api-Key` header or `?apikey=` query param against the stored host_config API key.

### auth.go

```go
package api

func apiKeyAuth(hostConfigStore hostconfig.Store) func(http.Handler) http.Handler {
    return func(next http.Handler) http.Handler {
        return http.HandlerFunc(func(w http.ResponseWriter, r *http.Request) {
            key := r.Header.Get("X-Api-Key")
            if key == "" {
                key = r.URL.Query().Get("apikey")
            }
            if key == "" {
                http.Error(w, `{"message":"Unauthorized"}`, http.StatusUnauthorized)
                return
            }
            hc, err := hostConfigStore.Get(r.Context())
            if err != nil || hc.APIKey != key {
                http.Error(w, `{"message":"Unauthorized"}`, http.StatusUnauthorized)
                return
            }
            next.ServeHTTP(w, r)
        })
    }
}
```

Apply to all `/api/v3/*` routes except `/ping`.

Tests: valid key → 200, missing key → 401, wrong key → 401.

Commit: `feat(api): add API key authentication middleware`

## Task 7 — Mount v3 subrouter + update server.go

Create `internal/api/v3/v3.go` that mounts all v3 routes on a chi subrouter:

```go
func Mount(r chi.Router, deps Dependencies) {
    r.Route("/api/v3", func(r chi.Router) {
        r.Use(apiKeyAuth(deps.HostConfig))

        series := &seriesHandler{...}
        r.Get("/series", series.list)
        r.Get("/series/{id}", series.get)
        r.Post("/series", series.create)
        r.Put("/series/{id}", series.update)
        r.Delete("/series/{id}", series.delete)

        // ... all other resources
    })
}
```

Update `internal/api/server.go` to pass the deps and call `v3.Mount`.

`Dependencies` is a struct holding all the stores and services the v3 handlers need — avoids a 20-parameter constructor.
